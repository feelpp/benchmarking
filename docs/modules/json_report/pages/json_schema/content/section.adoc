= Content Node: Section

Content is built using a list of *Nodes* under the root `content` field. The `SectionNode` is the primary way to structure the report hierarchically.

== SectionNode Schema

|===
|Field|Type|Description|Default Value

|`type`|string|Must be set to `"section"`.|`"section"`
|`title`|string|The title of the section (becomes an AsciiDoc heading).|*Required*
|`content`|array of objects|A recursive list of other content nodes, including nested `SectionNode`s.|*Required*
|===

=== Example

[source,json]
----
{
  "content": [
    {
      "type": "section",
      "title": "Introduction",
      "content": [
        // Text, Tables, Images, etc.
      ]
    },
    {
      "type": "section",
      "title": "Detailed Results",
      "content": [
        {
          "type": "section",
          "title": "Timing Analysis",
          "content": [
            // Plots and Tables
          ]
        }
      ]
    }
  ]
}
----
